# AI Development Notes

## AI Usage Summary

This document outlines how AI tools were used in building this application and what was manually verified.

## LLM Provider & Model Used

### Primary LLM: **Anthropic Claude (Claude Sonnet 4)**

**Why Claude was chosen:**

1. **Superior Code Generation**: Claude Sonnet 4 excels at generating clean, well-structured TypeScript/React code with proper typing
2. **Long Context Window**: Can handle entire file contents and complex multi-file codebases
3. **Following Instructions**: Excellent at adhering to specific requirements and coding patterns
4. **JSON/Structured Output**: Reliable for generating insights in consistent formats
5. **API Reliability**: Anthropic's API has good uptime and rate limits suitable for this use case
6. **Cost-Effective**: Good balance of performance and cost for a portfolio/demo project

**Model Details:**
- Model: `claude-sonnet-4-20250514`
- Used for: Generating insights, analyzing CSV data, answering follow-up questions
- Max tokens: Configured per request type (1000-4000 tokens)

## What AI Was Used For

### 1. Code Generation & Architecture (70% AI-assisted)
- **React Components**: Initial scaffolding and structure
  - AppShell, ReportPanel, ChatPanel, ChartPanel components
  - Used Claude to generate base components with TypeScript types
  - **Manual Review**: Checked prop types, ensured proper React patterns, removed unused code
  
- **State Management**: Zustand store setup
  - AI generated initial store structure
  - **Manual Review**: Verified state mutations, ensured proper typing, added missing actions
  
- **TypeScript Types**: Interface definitions
  - AI created type definitions from requirements
  - **Manual Review**: Checked for type safety, added missing optional properties

### 2. UI/UX Implementation (60% AI-assisted)
- **Tailwind CSS Styling**: 
  - AI suggested utility class combinations
  - **Manual Review**: Adjusted spacing, colors, responsive breakpoints
  
- **Dark/Light Theme**:
  - AI generated theme switcher and CSS variables
  - **Manual Review**: Tested color contrast, fixed edge cases in light mode
  
- **Chart Integration**:
  - AI provided Recharts implementation examples
  - **Manual Review**: Customized tooltips, fixed TypeScript errors, adjusted sizing

### 3. Data Processing Logic (50% AI-assisted)
- **CSV Parsing**: 
  - AI suggested PapaParse and SheetJS libraries
  - **Manual Review**: Added error handling, tested with various file formats
  
- **Column Type Inference**:
  - AI generated algorithm for detecting numeric/categorical columns
  - **Manual Review**: Tuned thresholds, added edge case handling
  
- **Data Aggregation for Charts**:
  - AI created initial aggregation logic
  - **Manual Review**: Fixed null handling, optimized performance

### 4. Backend API (40% AI-assisted)
- **Express Server Setup**:
  - AI generated boilerplate Express code
  - **Manual Review**: Added CORS configuration, error middleware
  
- **Anthropic API Integration**:
  - AI provided SDK usage examples
  - **Manual Review**: Implemented retry logic, added rate limiting
  
- **Health Check Endpoints**:
  - AI created basic health check structure
  - **Manual Review**: Added LLM connection testing, response time tracking

## Development Process

### Typical Workflow:
1. **Define Requirements** → Write clear specification in plain English
2. **AI Generation** → Use Claude to generate initial code
3. **Manual Review** → Read every line, understand logic
4. **Testing** → Test functionality manually
5. **Refinement** → Modify AI-generated code based on testing
6. **Integration** → Ensure new code works with existing codebase

### Example: Report History Dropdown

**AI Prompt:**
> "Create a dropdown component that shows the last 5 saved reports. Each item should show filename, timestamp, and row count. Clicking loads the report. Include a delete button on hover."

**AI Output:** Initial component with basic structure

**Manual Changes:**
- Added TypeScript types for SavedReport
- Implemented "click outside to close" functionality
- Added loading state handling
- Fixed z-index layering issues
- Improved accessibility (keyboard navigation)
- Added animations and transitions
